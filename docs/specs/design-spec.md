## 3. Domain Model

All entities in `com.wtechitsolutions.domain`. Schema auto-generated by Hibernate (`ddl-auto: create-drop`).

### Entities

**Account**
- `id`, `accountNumber`, `iban`, `bankCode`, `currency`, `balance`, `holderName`, `createdAt`

**Transaction**
- `id`, `accountId` (FK), `reference`, `amount`, `type` (CREDIT/DEBIT), `description`, `valueDate`, `entryDate`, `createdAt`

**BankingStatement**
- `id`, `accountId` (FK), `openingBalance`, `closingBalance`, `statementDate`, `sequenceNumber`, `createdAt`

**BenchmarkMetrics**
- `id`, `jobExecutionId`, `library` (Library enum), `fileType` (FileType enum)
- `throughputRps`, `generationDurationMs`, `parseDurationMs`, `memoryUsedBytes`
- `batchDurationMs`, `chunkDurationMs`, `cpuUsagePct`
- `successRate`, `failedCount`, `symmetryRate`, `recordsProcessed`, `timestamp`

### Data Generation

`DomainDataGenerator` service seeded on `POST /api/domain/generate`:
- 20 accounts, 200 transactions, 10 statements per call
- Returns unique `operationId` per generation run

## 5. Strategy Pattern

### Interface

```java
public interface FileGenerationStrategy {
    String generate(List<Transaction> transactions, List<Account> accounts);
    List<Transaction> parse(String fileContent);
    FileType getFileType();
    Library getLibrary();
}
```

### 8 Implementations

| Class | FileType | Library |
|---|---|---|
| `CodaBeanIOStrategy` | CODA | BEANIO |
| `CodaFixedFormat4JStrategy` | CODA | FIXEDFORMAT4J |
| `CodaFixedLengthStrategy` | CODA | FIXEDLENGTH |
| `CodaBindyStrategy` | CODA | BINDY |
| `SwiftBeanIOStrategy` | SWIFT | BEANIO |
| `SwiftFixedFormat4JStrategy` | SWIFT | FIXEDFORMAT4J |
| `SwiftFixedLengthStrategy` | SWIFT | FIXEDLENGTH |
| `SwiftBindyStrategy` | SWIFT | BINDY |

### Resolution

`StrategyResolver` (`@Service`) holds a `Map<String, FileGenerationStrategy>` keyed by `"CODA_BEANIO"` etc., auto-populated via Spring injection of all strategy beans. Resolves at runtime with no `if/switch` chains.

### Parser Layer

Four wrappers in `com.wtechitsolutions.parser`, each owning library-specific setup:
- `BeanIOFormatter` — BeanIO `StreamFactory`, XML mapping files in `resources/beanio/`
- `FixedFormat4JFormatter` — fixedformat4j annotation-driven mapping
- `FixedLengthFormatter` — `name.velikodniy.vitaliy:fixedlength:0.15` reflection-based mapping
- `BindyFormatter` — Apache Camel Bindy `FixedLengthDataFormat`

Format models in `com.wtechitsolutions.parser.model`:
- `CodaRecord` — 128-char fixed-width per Febelfin spec (record types 0, 1, 2, 8, 9)
- `SwiftMtRecord` — MT940 field tags (`:20:`, `:25:`, `:28C:`, `:60F:`, `:61:`, `:86:`, `:62F:`)

## 6. Spring Batch Pipeline

### Job Structure

```
bankingFileGenerationJob (restartable)
└── fileGenerationStep  (chunk-size = 100)
    ├── DomainEntityItemReader      JpaCursorItemReader, saveState=true
    ├── FileGenerationItemProcessor resolves strategy via StrategyResolver, maps domain → record string
    └── FileOutputItemWriter        writes /output/<fileType>_<library>_<timestamp>.txt,
                                    persists file content + metadata to DB

Listeners:
├── BatchMetricsListener  (JobExecutionListener) — writes BenchmarkMetrics row on completion
└── ChunkTimingListener   (ChunkListener)        — measures per-chunk duration
```

### Job Parameters

`fileType` (FileType), `library` (Library), `operationId` (Long), `runTimestamp` (Instant) — unique job instance key enabling restartability.

### BatchJobService

Launches jobs via `JobLauncher`, exposes execution history via `JobExplorer`. `GET /api/batch/history` returns last 50 executions.

## 7. REST API

### Endpoints

| Method | Path | Request Body | Response |
|---|---|---|---|
| POST | `/api/domain/generate` | — | `GenerateDomainResponse` |
| POST | `/api/batch/generate` | `BatchJobRequest` | `BatchJobResponse` |
| GET | `/api/batch/history` | — | `List<BatchHistoryResponse>` |
| GET | `/api/benchmark/results` | — | `List<BenchmarkResultResponse>` |
| GET | `/actuator/health` | — | Spring standard |
| GET | `/actuator/info` | — | app name, version, start time |

### DTOs (Java Records)

```java
record BatchJobRequest(FileType fileType, Library library) {}

record BatchJobResponse(
    Long jobExecutionId, FileType fileType, Library library,
    String status, String fileContent, String fileName, Instant timestamp) {}

record GenerateDomainResponse(
    Long operationId, int accountsGenerated,
    int transactionsGenerated, Instant timestamp) {}

record BatchHistoryResponse(
    Long jobExecutionId, FileType fileType, Library library,
    String status, long durationMs, Instant startTime, Instant endTime) {}

record BenchmarkResultResponse(
    Long id, Long jobExecutionId, FileType fileType, Library library,
    double throughputRps, long generationDurationMs, long parseDurationMs,
    long memoryUsedBytes, double successRate, double symmetryRate, Instant timestamp) {}
```

### Error Handling

`@RestControllerAdvice` returns `ProblemDetail` (RFC 9457, Spring 6 built-in):
- 400 — invalid `FileType` or `Library` enum value
- 404 — job execution not found
- 500 — batch job failure

### Security

Actuator: only `/health` and `/info` publicly exposed. Swagger: `springdoc-openapi-starter-webmvc-ui` activated in `dev` profile only.

## 12. Key Constraints

- Swagger enabled in `dev` profile only
- Spring Batch jobs restartable (chunk state persisted in H2)
- Output files written to `/output/` (gitignored)
- All generated files reproducible: same input + params → byte-identical output
- Benchmark datasets: 100k+ transactions supported; < 1k records completes in < 5s
- Benchmark metrics exportable: CSV, JSON, Markdown
- No secrets, credentials, or API keys committed
- Max source line length: 120 chars
- All packages: `com.wtechitsolutions.*`
